GeoNetwork opensource V2.1.0 alpha 1 Date: 16-10-2006

GeoNetwork server release notes

Introduction

Welcome to GeoNetwork opensource Spatial Data Catalogue system. The server installation has been developed to allow the full configuration of the base properties of the system during installation. This makes the installation fairly complex, since network details have to be known during the installation process. For desktop installations, please use the desktop specific installer.

Installation

The server installer will work on all operating systems running Java. A Java Runtime Environment (JRE) 1.5 has to be installed first.
You can start the installer from the command line/ terminal window using the following command:

java -jar geonetwork-intermap-2.1.0_alpha_1-install.jar

Requirements

Java

A Java Runtime Environment (JRE) 1.5 Standard Edition (download from http://java.sun.com) is required to run GeoNetwork opensource. See the Troubleshooting sections when normal installation or start up fails.

Web server

The installer comes with an embedded Jetty web server. You can use this server for testing. When moving to production you should use Jakarta Tomcat instead (see post-install notes).

You can open the GeoNetwork opensource homepage in a web browser by using the following URL http://<your server URL>:<your port>/geonetwork. If you install on your local machine using port 8080 (default), you could use http://localhost:8080/geonetwork

Database system

GeoNetwork desktop comes with an internal DBMS server, the McKoi SQL database.
You can use this DBMS for testing but it should not be used for server deployment. If you plan to use another DBMS please select that DBMS during the installation process and make sure that the DBMS is already up and running. Oracle and MySQL are currently supported
You can also configure the DBMS manually at a later stage, although this is more complex (See the community website for details)
